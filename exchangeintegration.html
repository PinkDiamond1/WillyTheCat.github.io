Integration of BitCash on your exchange

BitCash uses stealth addresses and it needs to check if an incoming transaction matches one of the addresses in the wallet.

This means that the node will significantly slow down when you add more and more stealth addresses. Also a rescan with thousands of addresses will be nearly impossible, because it would be to time consuming.

To solve this problem, we introduced a new kind of address for use on the exchanges, which acts like a normal Bitcoin address. You can create it with the RPC command: 
getnewaddress "" nonprivate
So you add the “nonprivate” as 3. parameter. Be sure that you do not have many stealth addresses in your wallet.dat file. If you already have many stealth address in your wallet.dat file, you should start over with a new wallet.dat file and only create this special kind of address. You should only use this kind of address for your exchange.

BitCash Dollar and BitCash Gold

We currently have 3 currencies on the blockchain. BitCash, BitCash Dollar and BitCash Gold. To receive BitCash Dollar, put a dollar@ in front of the deposit address. To receive BitCash Gold, put a gold@ in front of the deposit address.

We introduced special RPC commands for the use of BitCash Dollar and BitCash Gold. For example can use sendtoaddress if you are only using BitCash. If you want to use all 3 currencies, you can use sendtoaddressfromcurrency instead. You can specify the currency as first parameter (0 = BitCash, 1 = BitCash Dollar, 2 = BitCash Gold).

Also you can use getbalanceforcurrency instead of getbalance. And you can use movecurrency instead of move.
